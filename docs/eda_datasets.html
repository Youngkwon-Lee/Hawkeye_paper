Score 3-4 상세 분석 — "VLM이 반드시 맞춰야 하는 것"

MDS-UPDRS Score 기준 (임상적 의미)

Score Label Gait 예시 Finger Tapping 예시 시각적 명확도
0 Normal 정상 보행 정상 속도, 일정한 진폭 기준점
1 Slight 약간의 느림, 미세한 arm swing 감소 한 번의 중단 또는 약간 느림 어려움 ← VLM Challenge
2 Mild 독립 보행 가능하나 명확한 이상 2회 이상 중단, 경미한 amplitude 감소 중간
3 Moderate 보조기구 필요, 현저한 느림 많은 hesitation, 진폭 명확히 감소 명확
4 Severe 혼자 못 걸음, 보조 필수 거의 수행 불가 매우 명확

Task Score 3 Score 4 합계 전체 대비 Test set 내
Finger Tapping 23 2 25 3.1% 6개 (test)
Gait 8 0 8 1.9% 2개 (test)
Hand Movement 23 5 28 3.3% 4개 (test)

VLM 평가 전략:

1. Score 3-4 = "Must-pass" items
→ VLM이 이것도 못 맞추면 기본 시각 인식 능력 부족 판정
→ MotorExam에서 "Foundation Gate" 역할 (이걸 통과해야 다음 레벨 의미 있음)

2. Score 0 vs 1 = "Expert-level" items
→ 인간 전문가도 어려운 구간 (inter-rater Kappa ~0.6-0.7)
→ VLM이 여기서 잘하면 진짜 clinical value 입증

3. Score 1 vs 2 = "Clinical threshold"
→ 치료 결정에 영향 (약물 조절 기준)
→ 가장 실용적 가치가 높은 구간

VLM 실험 시사점

Score별 VLM 난이도 예측

비교 시각적 차이 예상 VLM 정확도 임상 중요도
Normal(0) vs Severe(3-4) 매우 명확 90%+ (쉬움) 낮음 (누구나 구별)
Normal(0) vs Mild(2) 중간 60-75% 중간
Normal(0) vs Slight(1) 미세 40-60% 높음 (조기 발견)
Slight(1) vs Mild(2) 미세 35-55% 높음 (치료 결정)

Class Imbalance 대응 전략

Evaluation (MotorExam)

  • Stratified sampling: 각 score에서 동수 추출
  • Score 0: 30, Score 1: 30, Score 2: 30, Score 3: 전수 (8-23)
  • QWK (Quadratic Weighted Kappa) 사용 → imbalance에 강건
  • Score별 accuracy 따로 리포팅

Training (QA Set)

  • Score 0-1에서 다운샘플링
  • Score 3-4에서 augmentation (프롬프트 변형)
  • YouTube 수집 시 Score 3-4 우선 수집
  • TULIP healthy control로 Score 0 다양화

480p 해상도의 영향

PD4T는 480p (SD) → Finger Tapping에서 문제 가능

  • Gait: 전신 움직임 → 480p에서도 충분히 관찰 가능
  • Leg Agility: 다리 전체 → 480p OK
  • Hand Movement: 손 전체 → 480p 경계선
  • Finger Tapping: 손가락 끝 미세 움직임 → 480p 부족할 수 있음

→ TULIP (1920x1200)과 비교하면 해상도 영향 정량화 가능
→ 논문에서 "Resolution as a confounding factor" 분석 가능

작년 결과에서 얻은 핵심 인사이트

VLM이 잘하는 것

  • Gait (거시 운동): ChatGPT 80.5% (TULIP)
  • 전신 움직임은 VLM이 잘 인식
  • Reasoning 생성 가능 (XAI)
  • 웹 UI 프로토타입 구현 완료

VLM이 못하는 것

  • Finger Tapping (미세 운동): Qwen 11%, Gemma 33%
  • 480p 해상도에서 손가락 세부 동작 인식 어려움
  • F1 Score가 Accuracy보다 훨씬 낮음 → class imbalance 문제
  • Zero-shot만 시도 → Few-shot/RAG 미검증
작년 발표 Limitations에서 지적된 것:
  1. "전체 데이터 수(N)가 적고, 고점수/고중증 환자 데이터 부족"
  2. "GPU 자원 제한으로 VLM Full Fine-tuning 미수행"
  3. "Kinematic Feature가 Pose Estimation 정확도에 직접 의존"
  4. "RGB/Skeleton/VLM 중 단일 모델 의존 → 통합 해석 어려움"
